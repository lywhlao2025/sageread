# 测试流程与用例矩阵（前端）

> 目标：前端与后端协作的关键流程必须可回归、可追踪、可执行。所有提交前必须跑完本文件列出的测试。

## 覆盖范围
- 翻译：选中文本翻译、对话翻译、结果渲染
- 对话：多轮对话、引用/解释、重试、流式输出
- 数据导入：本地导入/解析、元信息写入、列表刷新

## 测试层级与策略
### 1) 单元测试（Vitest）
- 目标：纯逻辑模块可回归（utils/services/ai 相关转换）
- 最小要求：提交前必须通过
- 命令：
```bash
pnpm --filter app test
```

### 2) E2E 测试（Playwright）
- 目标：覆盖用户关键路径，确保前后端串联无回归
- 最小要求：提交前必须通过
- 命令：
```bash
pnpm --filter app test:e2e
```

## 关键用例矩阵（必须覆盖）
### 翻译
- 选中文本 -> 触发翻译 -> 看到流式输出 -> 结果可复制
- 翻译失败（超时/额度耗尽） -> UI 友好提示
- 翻译输出不进入对话 session/context（避免污染后续对话）

### 对话
- 新对话发起 -> 流式输出可见 -> 会话可继续
- 引用文本问答 -> 引用渲染正确 -> 结果可重试
- 中断/重试 -> 不产生空响应或重复消息

### 数据导入
- 导入本地文件 -> 元信息解析 -> 书籍出现在列表
- 非支持格式 -> 明确错误提示

## 提交前检查清单（必做）
- [ ] `pnpm --filter app test`
- [ ] `pnpm --filter app test:e2e`
- [ ] `scripts/integration-test.sh`（前后端联调冒烟）
- [ ] 关键路径手动抽检（至少翻译/对话/导入各 1 条）
